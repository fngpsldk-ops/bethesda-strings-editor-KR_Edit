Contributing
============

Development setup
-----------------

.. code-block:: bash

   git clone https://github.com/0xra0/bethesda-strings-editor
   cd bethesda-strings-editor
   pip install PySide6 requests
   pip install ruff pyright pytest   # dev tools

Run the application:

.. code-block:: bash

   python main.py

Run tests:

.. code-block:: bash

   QT_QPA_PLATFORM=offscreen python -m pytest tests/ -v

Lint and type-check:

.. code-block:: bash

   ruff check .
   pyright

Compile Ukrainian UI translation after editing ``gui/translations/uk_UA.ts``:

.. code-block:: bash

   ./scripts/compile_translations.sh

Pre-commit hooks
----------------

.. code-block:: bash

   pip install pre-commit
   pre-commit install

This installs the ruff lint + format hooks defined in
``.pre-commit-config.yaml``.  They run automatically on ``git commit``.

Ollama model
------------

The app requires a custom Ollama model named ``translategemma3-st``.
Create it from the provided ``Modelfile``:

.. code-block:: bash

   ollama create translategemma3-st -f Modelfile

The ``Modelfile`` references a local GGUF path — edit it to point to
your copy of the model before running the command.

PyInstaller build (local)
--------------------------

.. code-block:: bash

   # Compile translations first
   ./scripts/compile_translations.sh

   # Write a dev version tag
   echo "__version__ = 'dev'" > _version.py

   pyinstaller bethesda_strings_editor.spec

The bundle lands in ``dist/bethesda-strings-editor/``.

Release process
---------------

1. Commit all changes to ``main``.
2. Tag the commit: ``git tag v0.2.0 && git push origin v0.2.0``
3. GitHub Actions builds Linux and Windows binaries, generates a
   changelog with git-cliff, and publishes a GitHub Release
   automatically.

The ``_version.py`` file is generated by CI from the git tag; the copy
in the repository is a placeholder (``__version__ = "dev"``).

Project layout
--------------

.. code-block:: text

   bethesda_strings/   Pure-Python parsing library (no Qt)
   gui/                PySide6 application layer
   tests/              pytest test suite
   benchmarks/         Performance benchmarks
   scripts/            Helper scripts (translation compile, etc.)
   resources/          Icons and QSS stylesheet
   data/               Word lists for language detection
   aur/                Arch Linux AUR package (PKGBUILD)
   docs/               This documentation (Sphinx)
   .github/workflows/  CI: lint, test, release, docs

Code style
----------

- **Formatter**: ruff (``ruff format``)
- **Linter**: ruff with ``E``, ``W``, ``F`` rule sets
- **Type checker**: Pyright
- Lines longer than 88 characters are allowed (E501 is ignored) because
  many Qt stylesheet strings cannot be broken.

Commit messages
---------------

The changelog is generated from commit messages using
`git-cliff <https://git-cliff.org>`_.  Prefix messages with a verb so
they land in the right section:

- ``Add …`` → **Added**
- ``Fix …`` → **Fixed**
- ``Remove …`` / ``Delete …`` → **Removed**
- ``Update …`` / ``Improve …`` / ``Refactor …`` / ``Change …`` → **Changed**
- Everything else → **Other**
