1. Go to the project on Weblate (link TBD once the component is registered).
2. Pick your language and start translating strings directly in the browser.
3. Weblate automatically opens a PR when enough strings are done.

## Manual workflow

### 1. Prerequisites

```bash
pip install PySide6          # provides pyside6-lupdate and pyside6-lrelease
```

### 2. Open the .ts file in Qt Linguist

```bash
pyside6-linguist gui/translations/de_DE.ts
```

Qt Linguist shows each source string alongside a text field for the
translation. Green checkmark = approved; yellow = unfinished.

### 3. Compile and test

```bash
./scripts/compile_translations.sh
python main.py
# Open Settings → Appearance → Interface Language, select your language
```

### 4. Submit a pull request

Commit only the `.ts` source file — `.qm` binaries are gitignored and
compiled automatically by CI:

```bash
git add gui/translations/de_DE.ts
git commit -m "Add German translation (de_DE)"
```

## Translation rules

### Preserve placeholders

Qt uses `%1`, `%2`, … for runtime values. Never translate or remove them:

| Source | ✅ Correct | ❌ Wrong |
|--------|-----------|---------|
| `Loaded %1 strings` | `%1 Strings geladen` | `Geladen Strings` |

### Preserve game tags

Strings marked **Context: game content** may contain Bethesda markup —
do **not** translate these in the UI itself; they pass through to the AI.

### Keep accelerators

Menu items use `&` as an accelerator prefix. Keep it, but move it to a
letter that exists in your language:

| Source | ✅ | ❌ |
|--------|----|----|
| `&File` | `&Datei` | `Datei` |

### Preserve formatting

Match the capitalization style of the source string:
- Title Case labels → Title Case in translation
- Sentence case descriptions → Sentence case

### Untranslatable strings

Leave these **empty** (they will fall back to English):
- Log messages (not shown in the UI)
- Internal identifiers / enum names
- Error codes like `MISSING_TAGS`, `RUSSIAN_LEAK`

## Adding a new language

1. Copy the Ukrainian source as a starting template:

```bash
cp gui/translations/uk_UA.ts gui/translations/ja_JP.ts
```

2. Edit the `language` attribute at the top of the new file:

```xml
<TS version="2.1" language="ja_JP">
```

3. Clear all `<translation>` elements (set them to `type="unfinished"`).

4. Add the locale to `_UI_LANGUAGES` in `gui/settings_dialog.py`:

```python
("ja_JP", "Japanese", "日本語", False),
```

5. Open a PR — even partial translations (≥ 30 %) are welcome.

## File structure

```
gui/translations/
  uk_UA.ts    ← Ukrainian source (complete, reference)
  uk_UA.qm    ← compiled binary (committed)
  de_DE.ts    ← German skeleton (needs translations)
  de_DE.qm    ← compiled binary (committed after editing)
  …
```

`.ts` files are XML — human-readable, diff-friendly, version-control friendly. **Commit these.**  
`.qm` files are compiled binaries loaded at runtime — gitignored, regenerated by CI and `scripts/compile_translations.sh`.
