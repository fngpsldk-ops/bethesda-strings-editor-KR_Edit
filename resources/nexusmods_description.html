Desktop tool for translating Starfield localization files across 11 languages using a locally-running AI model or the Claude API. Reads the binary string formats used by the Creation Engine ([b].strings[/b], [b].dlstrings[/b], [b].ilstrings[/b], [b]BA2 archives[/b]) and non-localized ESP/ESM plugins, sends text to Ollama or Claude, and writes the translated files back in the original format — ready to drop into your mod or game data folder. Designed around the real localization workflow: integrates with xTranslator XML files, protects thousands of Starfield-specific proper nouns from mistranslation, remembers every string you've already approved, and flags quality issues automatically so you can review or re-translate them one by one. [heading]Installation[/heading] [list=1] [*]Download the zip for your platform from the [b]Files[/b] tab [*]Extract anywhere — no Python or Qt install needed, fully self-contained [*]Install [url=https://ollama.com]Ollama[/url] and start it:[code]ollama serve[/code] [*]Pull the translation model (only needed once):[code]ollama pull 0xra/bethesda-translate ollama cp 0xra/bethesda-translate translategemma3-st[/code] [*]Run [b]bethesda-strings-editor[/b] (Linux) or [b]bethesda-strings-editor.exe[/b] (Windows) [/list] [b]Arch Linux (AUR):[/b] [code]yay -S bethesda-strings-editor-bin[/code] [b]Optional — AI quality checker model:[/b] [code]ollama pull 0xra/bethesda-qc ollama cp 0xra/bethesda-qc qcgemma4-st[/code] [b]Optional — Claude API backend:[/b] Enter your Anthropic API key in Settings → Claude AI. No local GPU required. [heading]Ollama Models[/heading] Both models are published on the Ollama hub and can be pulled directly — no local GGUF file needed.
