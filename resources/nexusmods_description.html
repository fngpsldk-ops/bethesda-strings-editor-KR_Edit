Bethesda Strings Editor Desktop tool for translating Starfield localization files across all 11 supported languages using a locally-running AI model. Reads the binary string formats used by the Creation Engine (.strings, .dlstrings, .ilstrings), BA2 archives, and non-localized ESP/ESM plugins, sends text to Ollama, and writes the translated files back in the original format — ready to drop into your mod or game data folder. Designed around the real localization workflow: integrates with xTranslator XML files, protects thousands of Starfield-specific proper nouns from mistranslation, remembers every string you've already approved, and flags quality issues automatically so you can review or re-translate them one by one. Every source→target language pair has a dedicated AI prompt with register rules, script conventions, and native translation examples baked in. Installation Download the zip for your platform from the Files tab Extract anywhere — no Python or Qt install needed, fully self-contained Install Ollama and start it: ollama serve Create the translation model (only needed once): ollama create translategemma3-st -f Modelfile Run bethesda-strings-editor (Linux) or bethesda-strings-editor.exe (Windows) Arch Linux (AUR): yay -S bethesda-strings-editor-bin Supported Languages All 9 official Starfield languages plus Russian and Ukrainian for xTranslator-style workflows: English · German · Spanish · French · Italian · Japanese · Polish · Portuguese (Brazil) · Chinese (Simplified) · Russian · Ukrainian Features Translation Parallel AI translation — configurable concurrency, 10 simultaneous requests by default Language-pair prompts — dedicated system prompt for every source→target combination with register rules, script conventions (Japanese polite forms, Chinese simplified terminology), and native examples Translation memory — approved strings are never sent to the model again Persistent cache — SHA-256 keyed, survives sessions, up to 50,000 entries Term protector — 8,000+ Starfield names, places, and UI labels swapped to placeholders before AI sees the text and restored afterward; proper nouns are never mistranslated Newline and whitespace restoration — when the model collapses a multi-line string into a single paragraph, the output is re-split proportionally and per-line indentation is restored from the original Mixed-script repair — stray Latin characters inside Cyrillic words are corrected automatically Glossary system — CSV/TBX/JSON import-export, in-app editor, automatic injection into AI prompts File Support .strings — null-terminated binary format .dlstrings / .ilstrings — length-prefixed binary format BA2 archives — reads and writes Starfield v2 and Fallout 4 v1 BA2 archives (GNRL type, zlib-compressed); picker dialog for multi-entry archives ESP/ESM plugins — extracts and writes back translatable fields from non-localized plugins directly xTranslator SST XML — import/export compatible with existing xTranslator projects Auto-detects encoding — UTF-8 BOM, UTF-8, CP1251, CP1252, CP1250 (Polish), GBK; per-file override Drag-and-drop file loading with format validation Review Workflow Quality checker — 20+ automated checks: missing/extra game tags (, [PLYR], %s), truncated AI output, suspicious length ratios, newline count mismatches, source-language leakage in Cyrillic output, AI artifact prefixes, encoding failures, and glossary mismatches One-click auto-fix for common issues (whitespace, capitalisation, character substitution, missing newlines); one-click re-translate with a per-string hint for AI issues Consistency checker — finds identical source strings translated differently; pick a canonical form and batch-replace Version diff — compare two game versions and migrate unchanged translations automatically; export CSV/HTML reports Vim-style keyboard navigation, command palette (Ctrl+K), customizable shortcuts UI Dark / Light / High-Contrast / Catppuccin themes plus custom QSS file support Ukrainian interface localization Drag-and-drop file open Status bar with live progress, translated count, and ETA during AI batches NexusMods upload — upload release archives directly to your mod page from inside the app (File → Upload to NexusMods…) Requirements Ollama must be running locally when you use AI translation features. The editor calls it over HTTP on localhost:11434 . A GGUF model compatible with Ollama. The included Modelfile points to translategemma-27b-it.Q4_K_M.gguf — edit the path to match your local model file before creating the model. Minimum hardware: 16 GB RAM recommended for the 27B Q4 model. Smaller GGUF models work too — just edit the Modelfile to point to a different one. The standalone builds include everything else (Python runtime, Qt libraries). No separate install needed.
