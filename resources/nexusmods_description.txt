Desktop tool for translating Starfield localization files across 11 languages using a locally-running AI model or the Claude API.

Reads the binary string formats used by the Creation Engine ([b].strings[/b], [b].dlstrings[/b], [b].ilstrings[/b], [b]BA2 archives[/b]) and non-localized ESP/ESM plugins, sends text to Ollama or Claude, and writes the translated files back in the original format — ready to drop into your mod or game data folder.

Designed around the real localization workflow: integrates with xTranslator XML files, protects thousands of Starfield-specific proper nouns from mistranslation, remembers every string you've already approved, and flags quality issues automatically so you can review or re-translate them one by one.

[heading]Installation[/heading]

[list=1]
[*]Download the zip for your platform from the [b]Files[/b] tab
[*]Extract anywhere — no Python or Qt install needed, fully self-contained
[*]Install [url=https://ollama.com]Ollama[/url] and start it:[code]ollama serve[/code]
[*]Run [b]bethesda-strings-editor[/b] (Linux) or [b]bethesda-strings-editor.exe[/b] (Windows)
[/list]

[b]Arch Linux (AUR):[/b][code]yay -S bethesda-strings-editor-bin[/code]

[b]Optional — AI quality checker model (hub):[/b][code]ollama pull 0xra/bethesda-qc
ollama cp 0xra/bethesda-qc qcgemma4-st[/code]

[b]Optional — Claude API backend:[/b] Enter your Anthropic API key in Settings → Claude AI. No local GPU required.

[heading]Ollama Models[/heading]

The QC model is published on the Ollama hub. Translation models are built from local GGUF files using the included Modelfiles.

[list]
[*][url=https://ollama.com/0xra/bethesda-qc][b]0xra/bethesda-qc[/b][/url] — Gemma 4 E4B fine-tuned specifically for Bethesda string quality checking. Detects 16 issue codes and returns structured VERDICT blocks with AUTOFIX/RETRANSLATE recommendations.
[/list]

[heading]Features[/heading]

[b]Translation[/b]
[list]
[*][b]Parallel AI translation[/b] — configurable concurrency, 10 simultaneous requests by default
[*][b]Claude API backend[/b] — use Haiku 4.5, Sonnet 4.6, or Opus 4.8 as a drop-in alternative to Ollama; no local GPU needed
[*][b]Language-pair prompts[/b] — dedicated system prompt for every source→target combination with register rules, script conventions, and native examples
[*][b]Translation memory[/b] — approved strings are never sent to the model again
[*][b]Persistent cache[/b] — SHA-256 keyed, survives sessions, up to 50,000 entries
[*][b]Term protector[/b] — 8,000+ Starfield names, places, and UI labels swapped to placeholders before the AI sees the text; proper nouns are never mistranslated
[*][b]Glossary system[/b] — CSV/TBX/JSON import-export, in-app editor, automatic injection into AI prompts
[*][b]Named sessions[/b] — save and restore search/filter state across sittings ([b]Ctrl+Shift+N[/b] / [b]Ctrl+Shift+S[/b])
[*][b]Macro recorder[/b] — record and replay sequences of edit operations ([b]Ctrl+M[/b])
[/list]

[b]File Support[/b]
[list]
[*][b].strings[/b] — null-terminated binary format
[*][b].dlstrings / .ilstrings[/b] — length-prefixed binary format
[*][b]BA2 archives[/b] — read and write Starfield v2 and Fallout 4 v1 BA2 archives (GNRL/zlib); picker dialog for multi-entry archives
[*][b]ESP/ESM plugins[/b] — extracts and writes back translatable fields from non-localized plugins directly
[*][b]xTranslator SST XML[/b] — import/export compatible with existing xTranslator projects
[*][b]Encoding detection[/b] — auto-detects UTF-8 BOM, UTF-8, CP1251, CP1252, CP1250 (Polish), GBK; per-file override
[*][b]Drag-and-drop[/b] file loading with format validation
[/list]

[b]Quality Assurance[/b]
[list]
[*][b]Quality checker[/b] — 20+ automated checks across all 11 languages: missing/extra game tags, truncated AI output, suspicious length ratios, source-language leakage, English leak, newline mismatches, AI artifact prefixes, encoding failures, CJK script coverage, and glossary mismatches
[*][b]AI quality model[/b] (qcgemma4-st) — fine-tuned Gemma 4 E4B that checks translations and returns structured verdicts with AUTOFIX/RETRANSLATE recommendations
[*][b]Hunspell spell-check[/b] — optional per-language spell checking using system dictionaries; install with [b]pacman -S hunspell-uk[/b] or [b]apt install hunspell-uk[/b]
[*][b]One-click auto-fix[/b] for mechanically correctable issues; one-click re-translate with a per-string hint for AI issues
[*][b]Consistency checker[/b] — finds identical source strings translated differently; pick a canonical form and batch-replace ([b]Ctrl+Alt+K[/b])
[*][b]Ukrainian gender checker[/b] — detects adjective/noun gender agreement mismatches using a Ukrainian noun gender dictionary ([b]Ctrl+Alt+G[/b])
[*][b]ти/ви register checker[/b] — detects mixed formal/informal address within a file ([b]Ctrl+Alt+R[/b])
[*][b]Version diff[/b] — compare two game versions and migrate unchanged translations; export CSV/HTML reports
[/list]

[b]NexusMods Integration[/b]
[list]
[*][b]Mod browser[/b] — search NexusMods directly from the app (File → Browse NexusMods…); card-grid results with thumbnails, endorsements, and download counts
[*][b]Free-user downloads[/b] — downloads work without a Premium subscription by reusing your active browser session (Firefox or Chromium); no NXM handler or manual clicks needed
[*][b]Archive extraction[/b] — automatically extracts .esp/.esm/.esl files from .zip, .7z, and .rar mod archives and opens them directly in the editor
[*][b]Translation Memory import[/b] — download a translation mod from NexusMods and load it as TM in one click; known strings are pre-filled and skipped by AI
[*][b]Mod upload[/b] — upload release archives directly from inside the app using the NexusMods v3 multipart upload API (File → Upload to NexusMods…)
[/list]

[b]UI / Workflow[/b]
[list]
[*]Vim-style keyboard navigation, command palette ([b]Ctrl+K[/b]), customizable shortcuts
[*][b]F7[/b] → next untranslated · [b]Ctrl+Enter[/b] → approve · [b]Ctrl+R[/b] → reject
[*]Dark / Light / Slate / custom QSS themes
[*]Interface available in Ukrainian ✓, German, Spanish, French, Polish, Czech (community WIP)
[*][b]GPU monitor[/b] — live GPU%, VRAM usage, and temperature in the status bar; color-coded green/yellow/red (AMD and NVIDIA)
[*]Status bar with live progress, translated count, and ETA during AI batches
[*][b]Focus mode[/b] — full-screen overlay showing one string at a time for distraction-free translation
[*][b]Dialogue tree viewer[/b] — visualizes Quest → Topic → Response hierarchy from an ESP/ESM file as an interactive tree ([b]Ctrl+Alt+D[/b])
[*][b]Lore RAG[/b] — import lore snippets that are injected into AI prompts for contextually accurate translation
[/list]

[heading]Requirements[/heading]

[url=https://ollama.com]Ollama[/url] must be running locally when you use AI translation features ([b]localhost:11434[/b]). Alternatively, use the [b]Claude API[/b] backend — no local GPU required, just enter your Anthropic API key in Settings.

[b]Minimum hardware:[/b] 8 GB VRAM recommended for the Gemma 3 12B Q4 translation model. The standalone builds include everything else (Python runtime, Qt libraries) — no separate install needed.
